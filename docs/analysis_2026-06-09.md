- **원인 분석**:
  - `stock_send.yml` 스케줄러는 매일 **08:00 KST**에 실행되도록 되어 있습니다.
  - 해당 스케줄러가 실행될 때 최신 리포트(.md)를 확인하는데, 당일 주식 리포트 `stock_2026-06-08.md`는 **09:45 KST**에 뒤늦게 머지되어 push되었습니다.
  - 이로 인해 스케줄러 실행 시점(08:00)에는 6/8 리포트가 존재하지 않아, 4일 전인 6/5 리포트를 탐색했습니다.
  - 발송 스크립트(`send_email.py`) 내의 안전 장치로 "최근 리포트 생성일이 3일 이내인 경우에만 발송"하는 로직이 있었기에, 4일 경과된 리포트는 전송을 건너뛰었습니다(skip).
- **조치 사항 및 교훈**:
  - 주식 리포트의 머지 및 push 시간이 아침 8시 이전으로 관리되어야 정상 발송됩니다. 파이프라인 상의 주식 리포트 생성 스케줄과 머지 루틴 일정을 조율해야 합니다.

### 2-3. Gemini 503 과부하 에러 완화를 위한 모델 폴백(Fallback) 기능 구현 완료
- **현상**: 기본 분석 모델인 `gemini-3.5-flash`가 일시적 트래픽 집중이나 API 과부하로 인해 503 Service Unavailable 및 429 Rate Limit 에러를 빈번히 뱉으며 리포트 생성에 실패함.
- **개선 사항**:
  - 환경변수로 설정 가능하도록 `config/settings.py`에 `GEMINI_MODEL_FALLBACK` 옵션 정의 (기본값: `gemini-2.5-flash`).
  - 뉴스 및 주식 분석 모듈(`core/news/analyzer.py`, `core/stock/analyzer.py`)에 에러 처리를 도입하여, `gemini-3.5-flash`로 총 3회 재시도 실패 시, 자동으로 백업 모델(`gemini-2.5-flash`)로 3회 추가 재시도를 수행하는 유연한 폴백 로직을 구현하여 메인 브랜치에 병합하였습니다.

## 3. 카드뉴스 이미지 구현 및 SNS 연동 현황 점검

현재 카드뉴스 자동화는 **HTML 템플릿 생성 → 이미지 파일 렌더링 → SNS 자동 포스팅**으로 이어지는 파이프라인의 핵심 코드가 모두 구축되어 있으나, 일부 외부 플랫폼의 API 인증 인프라가 미설정 상태입니다.

### 3-1. 카드뉴스 HTML 생성 ([build_cardnews.py](file:///c:/ai/dailynews/scripts/build_cardnews.py))
- **현황**: **구현 완료**
- **기능**:
  - 뉴스, AI이슈, 주식 3개 채널의 원시 데이터(JSON)를 취합하여 가로 1080px, 세로 1080px 크기에 특화된 슬라이드형 HTML(`publish/cardnews/{channel}/{date}.html`) 및 인덱스(`data.json`)를 생성합니다.
  - URL의 `?card=N` 파라미터에 따라 자바스크립트로 해당 순번의 슬라이드만 활성화하여 렌더링되도록 구현되어 있어 이미지 추출에 용이합니다.

### 3-2. 이미지 스크린샷 렌더링 ([generate_cardnews_images.py](file:///c:/ai/dailynews/scripts/generate_cardnews_images.py))
- **현황**: **구현 완료 (듀얼 백엔드 구성)**
- **동작 방식**:
  - **Playwright 백엔드 (기본)**: 로컬 HTTP 서버를 백그라운드에 구동하고 헤드리스 크로미움(Chromium) 브라우저로 카드뉴스 HTML의 슬라이드 번호별(`?card=0, 1, 2...`) 화면에 접속하여 고품질 PNG 스크린샷을 찍습니다. 구글 웹 폰트(Noto Sans KR)가 정상 반영됩니다.
  - **Pillow 백엔드 (폴백)**: Playwright나 브라우저 드라이버를 로컬에 기동할 수 없는 네트워크 차단(SSL 복호화 프록시 등) 및 설치 제한 환경을 우려해, Python의 PIL 라이브러리를 사용해 직접 그래디언트 배경 위에 요약 텍스트와 레이아웃을 덧씌우는 자체 백업 렌더링 로직도 완벽 구현되어 있습니다.

### 3-3. 멀티 SNS 플랫폼 발송 ([post_cardnews.py](file:///c:/ai/dailynews/scripts/post_cardnews.py) 및 [post_instagram.py](file:///c:/ai/dailynews/scripts/post_instagram.py))
- **현황**: **부분 완성 (일부 설정 및 연동 미비)**
- **채널별 상태**:
  1. **텔레그램 (Telegram) [완료]**:
     - 기존 `TELEGRAM_BOT_TOKEN` 및 `TELEGRAM_CHAT_ID`를 그대로 활용하여 카드뉴스 1080x1080 PNG 파일 묶음을 미디어 그룹(sendMediaGroup)으로 자동 업로드하고 하단에 원문 링크 버튼을 첨부하여 발송합니다.
  2. **인스타그램 (Instagram) [미완료 - 환경변수 미설정]**:
     - Meta Graph API v21.0를 기반으로 다중 이미지 카루셀(Carousel) 형태 포스팅을 게시하기 위한 4단계 API(개별 이미지 업로드 -> 폴링 체크 -> 카루셀 취합 -> 발행) 연동 코드는 구현되어 있으나, API 통신에 필요한 **`INSTAGRAM_ACCESS_TOKEN`** 및 **`INSTAGRAM_BUSINESS_ACCOUNT_ID`** 가 GitHub Secrets에 등록되지 않은 채 대기 중입니다.
  3. **트위터 / X [미완료 - 환경변수 및 패키지 설치 필요]**:
     - `tweepy` 모듈을 사용해 트위터 API v2로 업로드하는 코드가 구성되어 있으나, 트위터 개발자 계정의 API Key 및 OAuth Access Token (`TWITTER_API_KEY`, `TWITTER_API_SECRET`, `TWITTER_ACCESS_TOKEN`, `TWITTER_ACCESS_TOKEN_SECRET`)이 GitHub Secrets에 미설정 상태입니다.
- **워크플로우 연동 ([cardnews.yml](file:///c:/ai/dailynews/.github/workflows/cardnews.yml))**:
  - 데일리 뉴스 리포트, AI 이슈 리포트, 주식 빌드 완료 시 이벤트(`workflow_run`)를 받아 카드뉴스 빌드 및 PNG를 생성하여 자동으로 git repository에 push하고 `post_cardnews.py`를 구동하는 전체 파이프라인이 세팅 완료되었습니다. 현재 인스타그램/트위터 토큰 미비로 에러가 발생하더라도 파이프라인 자체가 깨지지 않도록 `continue-on-error: true`가 걸려있습니다.

## 4. 프로젝트 미완료 태스크 정리 (우선순위 순)

### 📌 우선순위 1: 구독자 관리 시스템 도입 (Phase 2 로드맵)
현재 이메일 발송 대상이 `RECIPIENT_EMAILS` 환경변수에 하드코딩되어 한정적입니다. 이를 클라우드 DB 구독 시스템으로 점진 이전해야 합니다.
- [ ] **Supabase DB 세팅**: `subscribers` 테이블 및 발송 이력을 기록할 `send_log` 테이블 생성 및 권한 설정.
- [ ] **Vercel Serverless API 엔드포인트 구현**:
  - `api/subscribe.py`: 이메일 등록 및 본인 인증 이메일 발송
  - `api/verify.py`: 이메일 링크 내 인증 토큰 검증 후 상태를 active로 업데이트
  - `api/unsubscribe.py` / `api/manage.py`: 수신 거부 및 구독 정보(채널별 뉴스/주식/AI이슈 개별 선택) 변경 관리
- [ ] **발송기 수정 (`core/shared/mailer.py`)**: 하드코딩된 이메일 목록 대신 Supabase DB에서 `active` 상태이고 해당 채널을 구독한 대상자 목록을 조회하여 대량 메일 발송을 처리하도록 변경.
- [ ] **구독 신청 폼 UI 개발**: Vercel 및 메인 웹사이트 상에 구독하기 폼 (`publish/subscribe.html`)을 추가하고 API 연동.

### 📌 우선순위 2: SNS 카드뉴스 자동 발송 기능 활성화 (Phase 3 로드맵)
- [ ] **인스타그램 API 연동 완료**: Meta for Developers에서 장기 사용 액세스 토큰(Long-lived Access Token, 60일 유효)을 발급하여 GitHub Secrets에 `INSTAGRAM_ACCESS_TOKEN` 및 `INSTAGRAM_BUSINESS_ACCOUNT_ID` 등록. (추후 토큰 자동 갱신 스크립트 작성 검토 필요)
- [ ] **트위터 API 연동 완료**: Twitter Developer App에서 API 키 및 OAuth 1.0a용 Access Token을 발급받아 GitHub Secrets에 관련 4개 변수 등록.
- [ ] **카드뉴스 파이프라인 발송 수동 검증**: `cardnews.yml`을 수동(`workflow_dispatch`) 실행하여 텔레그램 카드뉴스 수신 상태 및 타 SNS 전송 상태 최종 검증.

- [ ] **데이터 마이그레이션 스크립트 작성**: 기존 xlsx 엑셀 데이터를 Supabase PostgreSQL 테이블로 일괄 전송하고 무결성 검증하는 일회성 마이그레이션 코드 수행.
