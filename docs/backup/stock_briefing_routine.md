# 📊 주식 시황 브리핑 루틴 (v4)

> **실행 조건**: 스케줄러에 의해 자동 실행 (KST 16:30 이후로 별도 지정)
> **실행 환경**: Claude Code 클라우드 루틴
>   - repo가 clone된 컨테이너 내부에서 실행
>   - 파일 저장·git 명령은 모두 상대경로 사용
>   - GitHub push는 루틴에 연결된 repo 권한으로 처리 (별도 토큰 불필요)
>   - 수집 종목 관리: `.claude/config/watchlist.yaml` (루틴 프롬프트 수정 불필요)

---

## [Step 1: 데이터 수집]

### 1-1. 종목 목록 로드

Read tool로 `.claude/config/watchlist.yaml`을 읽어 수집할 티커 목록을 확인하라.
파일 구조: `indices` / `macro` / `us_indices` / `sectors` 4개 섹션으로 구성.
이후 모든 수집은 이 파일에 정의된 티커를 기준으로 한다.

### 1-2. PlayMCP UsStockInfo — 주가/지표 수집

watchlist.yaml에 정의된 **모든 티커**를 순서대로 `get_stock_info`로 수집하라.
- 각 티커는 **개별 호출**한다
- 호출 실패·응답 없음·null 값인 경우 해당 항목만 `"N/A"`로 기록하고
  **다음 티커로 반드시 계속 진행**한다
- 하나의 실패가 전체 루틴을 중단시켜서는 안 된다

### 1-3. PlayMCP NaverSearch — 뉴스 수집

아래 4개 쿼리로 뉴스를 수집하라.
각 쿼리 실패 시 해당 결과만 빈 배열로 처리하고 진행 계속.

| 쿼리 | 결과 수 | 목적 |
|------|---------|------|
| `"코스피 오늘 종가 시황"` | 5건 | 당일 지수 흐름 |
| `"외국인 기관 순매수 오늘"` | 3건 | 수급 동향 |
| `"국내 증시 테마주 오늘"` | 3건 | 섹터/테마 이슈 |
| `"미국 증시 영향 국내"` | 3건 | 글로벌 연계 이슈 |

- **읽기 실패**: 아래 2-2 내장 구조로 대체하고,
  리포트 최상단에 `<!-- 템플릿 로드 실패: 내장 구조 사용 -->` 기록

### 2-2. 리포트 내장 구조 (fallback)

### 2-3. 시장 온도계 판단 기준

아래 기준을 **우선순위 순서대로** 적용하여 하나를 선택하라.

**🔴 리스크오프** — 다음 중 2개 이상 해당:
- 코스피 -1.0% 이상 하락
- 원/달러 환율 1,400원 초과
- 미 10년물 금리 당일 상승 + 4.5% 초과
- S&P 500 -1.0% 이상 하락
- 섹터 전체 중 75% 이상 🔴

**🟢 리스크온** — 다음 중 2개 이상 해당:
- 코스피 +1.0% 이상 상승
- 원/달러 환율 1,350원 미만
- S&P 500 +1.0% 이상 상승
- 외국인 순매수 우세 (뉴스 기반 판단)
- 섹터 전체 중 75% 이상 🟢

**🟡 중립** — 위 두 조건 미충족 시

### 2-4. 섹터 방향 판단 기준

- 🟢: 섹터 내 tickers 평균 등락률 +0.5% 초과
- 🔴: 섹터 내 tickers 평균 등락률 -0.5% 미만
- 🟡: 그 외 (-0.5% ~ +0.5% 범위)

## [Step 4: Notion 등록 — Notion MCP]

> **데이터베이스 ID 고정**: `362fae73-9c2b-80c2-ba08-c6f31ae8f922`
> (매번 notion-search로 탐색하지 않는다)

### 4-1. 중복 체크

`notion-search`로 오늘 날짜(`YYYY-MM-DD`)가 제목에 포함된 페이지를 먼저 조회한다.
- **이미 존재**: `notion-update-page`로 기존 페이지를 업데이트한다
- **없음**: `notion-create-pages`로 신규 생성한다

### 4-2. 페이지 속성 매핑

| Notion 속성명 | 타입 | 입력값 |
|--------------|------|--------|
| `이름` | title | `[YYYY-MM-DD] 주식 시황 브리핑` |
| `날짜` | date | 오늘 날짜 (`YYYY-MM-DD`) |
| `시장온도` | select | `리스크오프` / `중립` / `리스크온` (이모지 제외) |
| `코스피` | number | 코스피 종가 숫자 (N/A면 입력 생략) |
| `원달러` | number | 원/달러 환율 숫자 (N/A면 입력 생략) |
| `핵심키워드` | multi_select | 키워드 ①~⑤ 텍스트 5개 |
| `summary` | rich_text | 종합 요약 200자 내외 |
| `리포트링크` | url | GitHub Pages 발행 URL (빌드 전이면 생략) |

### 4-3. 실패 처리

Notion API 오류 발생 시 오류 내용을 콘솔에 출력하고 **루틴은 계속 진행**한다.

## [Step 5: Git 커밋 + 푸시 — Bash tool]

```bash
TODAY=$(date +%Y-%m-%d)
git add reports/stock/
git commit -m "📊 Stock briefing ${TODAY}"
git push
```

push 결과(성공/실패)를 콘솔에 출력하라.
push 실패 시에도 루틴은 정상 종료한다.

> **GitHub Actions 연계**
> git push 성공 후 `stock_build.yml` 워크플로우가 자동 트리거되어
> 웹페이지 빌드 및 이메일 발송을 처리한다. 루틴에서 별도 실행 불필요.

## [실행 흐름 요약]

```
[스케줄러 자동 실행]
        │
        ▼
[Step 1] watchlist.yaml 로드 → PlayMCP 수집 (전체 티커 + 뉴스 4쿼리)
         └─ 개별 실패 → N/A 처리 후 계속
        │
        ▼
[Step 2] 리포트 작성
         ├─ 템플릿 로드 성공 → 템플릿 구조 사용
         └─ 템플릿 로드 실패 → 내장 구조 fallback
         └─ 섹터 테이블 → watchlist.yaml sectors 기준 자동 생성
        │
        ▼
[Step 3] Write tool → reports/stock/stock_YYYY-MM-DD.md 저장
        │
        ▼
[Step 4] Notion MCP → 중복 체크 후 create / update
         └─ 실패해도 루틴 계속 (비중단)
        │
        ▼
[Step 5] git add → commit → push
         └─ GitHub Actions stock_build.yml 자동 트리거
              ├─ 웹페이지 빌드
              └─ 이메일 발송
```
