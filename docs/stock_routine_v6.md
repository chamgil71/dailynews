# 📊 주식 시황 브리핑 루틴 v6

## 개요
매 거래일(월~금) 오후 9:25 KST에 자동 실행. 역할: 시장 데이터 수집 → LLM 분석 → MD 저장 → Git push. 배포(이메일·Notion·HTML)는 GitHub Actions `stock_build.yml`이 처리.

토요일 전용 "주간 종합 루틴"은 별도 문서(`weekly_routine_v1.md`) 참고.

## Step 1: 시장 지수·환율·금리·유가 수집

다음 티커 목록의 현재가와 등락률을 수집하라. MCP 도구: `UsStockInfo.get_stock_info`

수집 대상 (한 번에 하나씩, 순서대로):

- `^KS11` → 코스피
- `^KQ11` → 코스닥
- `KRW=X` → 원/달러 환율
- `^GSPC` → S&P 500
- `^IXIC` → 나스닥
- `^DJI` → 다우존스
- `^TNX` → 미 10년물 국채금리 (단위: %, bp 변동 계산)
- `CL=F` → WTI 유가

수집 결과 형식:


## Step 6: GitHub에 직접 Push (`mcp__github__push_files`)

⚠️ 웹 루틴은 세션 브랜치에서 실행되므로 `git push origin main`이 불가. 반드시 MCP 도구를 사용해 main에 직접 커밋한다.

- 도구: `mcp__github__push_files`
- 파라미터:
  - `owner`: chamgil71
  - `repo`: dailynews
  - `branch`: main
  - `message`: `"📊 주식 시황 YYYY-MM-DD"`
  - `files`: `[{ path: "reports/stock/stock_YYYY-MM-DD.md", content: <MD 전체 내용> }]`

push 성공 시 GitHub Actions `stock_build.yml`이 자동으로:
- HTML 빌드 및 GitHub Pages 배포
- 이메일 발송 (익일 08:00 KST, `stock_send.yml`)
- Notion 동기화
- 히스토리 업데이트

Step 6 완료 후 루틴 종료. Notion·이메일·텔레그램은 GitHub Actions에서 처리하므로 루틴에서는 실행하지 않는다.
