# 📅 주간 종합 루틴 v1

## 개요
매주 토요일 오전 09:00 KST에 자동 실행 (평일 루틴과 독립). 역할: 주간 데이터 취합 → LLM 분석 → 주간 MD 저장 → Git push.

평일 일일 루틴은 별도 문서(`stock_routine_v6.md`) 참고.

---

## Step 1: 주간 데이터 수집 (평일 5일치 MD)

`mcp__github__get_file_contents` 도구를 사용해 당주 월~금(5일) `reports/stock/stock_YYYY-MM-DD.md` 파일을 순서대로 가져온다.

- owner: `chamgil71`
- repo: `dailynews`
- branch: `main`
- 파일 경로 예시: `reports/stock/stock_2026-06-09.md` (월) ~ `reports/stock/stock_2026-06-13.md` (금)

각 MD에서 추출할 항목:
- 날짜
- 지수 데이터 (코스피·코스닥·S&P 500·나스닥 종가, 등락률)
- 섹터 요약 테이블 (2-1. 섹터 요약 — 대표종목 등락률)
- 핵심 키워드 TOP 5
- 시장 온도계

MD 파일이 없는 날(공휴일·휴장)은 건너뛰고, 수집된 날짜만으로 분석한다.

---

## Step 2: 주간 지수 변동 집계

수집한 5일치 데이터에서 주간 시가(월요일 종가)·주간 종가(금요일 종가)를 기준으로 주간 변동률을 계산한다.

집계 대상:
- 코스피 주간 변동률 (%)
- 코스닥 주간 변동률 (%)
- S&P 500 주간 변동률 (%)
- 나스닥 주간 변동률 (%)
- 원/달러 주간 변동 (금요일 종가 기준)
- WTI 유가 주간 변동 (금요일 종가 기준)

---

## Step 3: 섹터 주간 성과 집계

9개 섹터별로 대표종목의 일별 등락률을 합산해 주간 누적 성과를 도출한다.

- 상승 상위 3섹터 / 하락 상위 3섹터 식별
- 상위 1~2섹터: NaverSearch.search_news로 원인 뉴스 1~2건 검색
  - 쿼리 예: `"반도체 주가 이번주 상승 이유"`

---

## Step 4: 핫 테마 분석

5일치 핵심 키워드 TOP 5를 합산해 주간 빈도 순위를 도출한다.

- 상위 3개 테마 선정
- 각 테마에 대해 1~2회 추가 검색으로 배경 보완
  - 도구: `NaverSearch.search_news` 또는 `NaverSearch.search_webkr`
  - 쿼리 예: `"AI 반도체 주간 동향 2026-06-13"`

---

## Step 5: 차주 주요 일정 수집

다음 주 시장에 영향을 줄 수 있는 주요 일정을 수집한다.

- 검색 도구: `NaverSearch.search_news`
- 쿼리 1: `"FOMC 다음주 일정 금리"` (필요 시)
- 쿼리 2: `"한국 금통위 일정 기준금리"`
- 쿼리 3: `"미국 경제지표 발표 일정 다음주"`

수집 항목: 날짜, 이벤트명, 예상 영향

---

## Step 6: LLM 종합 분석

수집한 데이터를 종합해 아래 항목을 작성하라:

- **주간 한줄 총평**: 이번 주 시장을 한 문장으로 요약
- **주간 온도계**: 🔴 과열 / 🟠 강세 / 🟡 중립 / 🟢 약세 / 🔵 침체 중 선택
- **주도 섹터 분석**: 상위 3섹터 선정 이유 및 다음 주 지속 가능성
- **핫 테마 요약**: 상위 3테마 각 2~3문장 설명
- **차주 전략 포인트**: 투자자 관점에서 주목할 사항 2~3개
- **리스크 요인**: 다음 주 주요 리스크 1~2개

---

## Step 8: GitHub에 직접 Push

⚠️ 웹 루틴은 세션 브랜치 제약으로 `git push origin main`이 불가. 반드시 MCP 도구를 사용한다.

- 도구: `mcp__github__push_files`
- 파라미터:
  - `owner`: chamgil71
  - `repo`: dailynews
  - `branch`: main
  - `message`: `"📅 주간 시황 종합 YYYY-MM-DD"`
  - `files`: `[{ path: "reports/stock/weekly_YYYY-MM-DD.md", content: <MD 전체 내용> }]`

push 성공 후 루틴 종료. 주간 리포트의 이메일·HTML 빌드는 별도 GitHub Actions 구현 전까지 수동 처리.
