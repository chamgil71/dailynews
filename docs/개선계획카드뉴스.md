### 1. HTML 상단에 핵심 이슈/키워드 3개 앵커 링크 제공
마크다운의 목차(TOC, Table of Contents) 기능을 활용하거나, AI가 직접 상단 요약본과 앵커 링크를 생성하도록 프롬프트를 수정하면 됩니다. 가장 쉬운 방법은 `markdown2` 라이브러리의 `toc` 익스텐션을 켜는 것입니다.

**수정 방법:** `scripts/build_site.py` 파일의 HTML 변환 부분에 `toc`를 추가하세요.
```python
# scripts/build_site.py의 build_report_page 함수 수정
html_body = markdown2.markdown(
    raw,
    extras=["tables", "fenced-code-blocks", "strike", "cuddled-lists", "toc", "header-ids"]
)
```
이렇게 하면 마크다운의 `##` 헤더들에 자동으로 ID가 부여됩니다. 그런 다음 AI 프롬프트에서 상단에 목차를 스스로 작성하도록 지시할 수 있습니다.

### 3. 이메일 본문에 접기/펴기 (Collapsible List) 적용
**이메일에서는 권장하지 않습니다.**
HTML의 `<details>`와 `<summary>` 태그를 이용한 접기/펴기 기능은 웹(웹브라우저)에서는 완벽하게 동작하지만, **Gmail, Outlook, 네이버 메일 등 대부분의 이메일 클라이언트에서는 보안 및 렌더링 엔진의 한계로 해당 태그를 무시하거나 항상 펼쳐진 상태로 강제 노출**시킵니다. 

이메일 본문이 지저분해지는 것을 방지하려면, 전체 기사 목록은 이메일에서 아예 빼거나 간소화하고 **"웹에서 전체 기사 보기" 버튼을 넣는 방식**이 훨씬 깔끔합니다.

### 4. 1번에서 만든 내용을 이메일로 발송
**가능합니다.** `core/mailer.py`는 이미 `markdown2`를 이용해 리포트를 HTML로 변환하여 메일로 쏘고 있습니다. AI가 작성한 이슈/키워드 요약과 대표 URL은 메일 본문에 그대로 잘 나타납니다. 

다만, 이메일 내에서의 앵커 링크(클릭 시 아래로 스크롤)는 모바일 앱(iOS 메일앱, 일부 안드로이드 메일앱) 환경에 따라 작동하지 않을 수 있다는 점은 감안하셔야 합니다. (웹메일에서는 잘 작동합니다.)

### 5. 추가 메일링 디자인 제안 및 개선안

현재 `core/mailer.py`의 이메일 HTML은 기본 텍스트 위주로 되어 있습니다. 가독성을 높이고 프로페셔널한 뉴스레터 느낌을 주기 위해 아래와 같은 디자인과 구조를 제안합니다.

* **요약 박스(Highlight Box) 도입:** 상단의 '핵심 이슈 TOP 3' 부분을 옅은 회색이나 파란색 배경의 박스 안에 배치하여 한눈에 들어오게 합니다.
* **긴 목록 과감히 생략 (클리핑 방지):** Gmail은 메일 용량이 102KB를 넘으면 하단을 잘라버리고 "메시지 전체 보기" 링크를 띄웁니다. 수십 개의 전체 수집 기사 목록은 과감히 생략하고 웹 링크로 유도하는 것이 좋습니다.
* **CTA(Call to Action) 버튼:** 웹사이트로 유도하는 둥근 버튼 형태의 링크를 하단에 배치합니다.

### 2. 파이썬 연동 로직 제안 (`scripts/build_site.py` 수정 방향)

`config/prompts.py`에서 정의된 "핵심 이슈 TOP 3"와 "주목할 트렌드" 텍스트를 파싱하여 위 HTML 구조에 끼워 넣는 로직이 필요합니다.

**추가해야 할 파싱 함수 예시:**

```python
def generate_card_news_html(analysis_text):
    """
    분석 텍스트에서 이슈 3개, 키워드 3개를 추출해 카드뉴스 HTML을 생성합니다.
    """
    # 1. 정규표현식 등으로 이슈와 키워드 파트를 분리 (현재 prompt 구조 활용)
    # 이슈 파트 추출 (1. **제목** — 내용)
    issues = re.findall(r'\d+\.\s+\*\*(.+?)\*\*\s*[—-]\s*(.+)', analysis_text)
    # 키워드 파트 추출 (- **키워드**: 내용)
    keywords = re.findall(r'-\s+\*\*(.+?)\*\*[:：]\s*(.+)', analysis_text)

    card_html = '<div class="card-news-container">'
    
    # 이슈 카드 생성 (최대 3개)
    icons = ["🔥", "📢", "💡"]
    for i, (title, content) in enumerate(issues[:3]):
        card_html += f"""
        <div class="news-card type-issue">
            <div class="card-header"><div class="icon">{icons[i]}</div><h3>{title}</h3></div>
            <div class="card-body">{content}</div>
            <div class="card-footer">핵심 이슈 TOP 3</div>
        </div>"""

    # 키워드 카드 생성 (최대 3개)
    k_icons = ["🏷️", "📊", "🎯"]
    for i, (title, content) in enumerate(keywords[:3]):
        card_html += f"""
        <div class="news-card type-keyword">
            <div class="card-header"><div class="icon">{k_icons[i]}</div><h3>#{title}</h3></div>
            <div class="card-body">{content}</div>
            <div class="card-footer">주목할 트렌드</div>
        </div>"""
        
    card_html += '</div>'
    return card_html
```

### 3. 디자인 개선 및 적용 팁

1.  **아이콘 자동화:** 제목에 '기술'이 포함되면 💻, '경제'면 📈가 나오도록 매핑 테이블을 만들어 `config/prompts.py`의 `CATEGORY_PROMPTS`와 연동
2.  **이메일 발송 시 주의:**  **웹사이트(GitHub Pages)**에서는 슬라이드로 보여주고, **이메일**에서는 카드를 위에서 아래로 세로로 나열하는 '반응형' 처리를 하는 것이 가장 좋습니다.
3.  **상단 제목 하단 내용:** 카드 내에서 `flex-direction: column`을 사용했으므로, 제목이 길어져도 하단 내용 영역이 밀리지 않도록 제목 영역에 `height`를 고정해 주는 것이 정렬에 유리합니다.
4.  **카드뉴스 폴더 생성:** 해당 카드 뉴스는 기존 폴더 하단에 cardnews/ 폴더에 별도 저정하고 메인페이지에 카드뉴스목록도 표시하고 이동할 수 있도록 한다. 

이 템플릿을 `templates/daily_report.md` 상단에 삽입하고 `scripts/build_site.py`에서 변환 시 해당 영역을 파싱된 HTML로 치환하도록 설정하면 프로젝트에 바로 적용 가능합니다.
