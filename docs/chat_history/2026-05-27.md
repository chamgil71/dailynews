* **작성 일자**: 2026-05-27

## 1. 주요 안건 및 문제 정의

### 🚨 핵심 문제
1. **배포 플랫폼 불일치 및 덮어쓰기 문제**:
   * Vercel과 GitHub Pages가 서로 다른 타이밍에 배포를 수행하여 내용이 일치하지 않는 문제 발생.
   * 특히, 피처(feature) 브랜치에서 `stock_build.yml`이 동작할 경우, 개발 중인 낡거나 불완전한 `publish/` 상태로 GitHub Pages를 덮어씌워 구버전으로 되돌아가는 현상 발생.
2. **주식 시황 요약 UI의 렌더링 버그**:
   * 주식 시황 요약 카드에서 줄바꿈이 무시되어 모든 내용이 한 줄로 뭉개져 출력됨.
   * 마크다운 볼드체 기호(`**`)가 평문 텍스트로 그대로 노출되어 시각적 결함 발생.

---

## 2. 해결 과정 및 설계 결정

### 🛠️ 배포 파이프라인 최적화 (오버엔지니어링 탈피)
* **초기 제안 (AI)**:
  * 배포 딜레이를 없애기 위해 배포만 100% 전담하는 독립적인 `deploy.yml` 워크플로우를 신설하는 아키텍처를 제시함.
* **사용자 피드백**:
  * 구조가 불필요하게 복잡해지는 것에 대해 지적하며, 본질적인 니즈인 **"Vercel과 GitHub Pages의 100% 실시간 동일한 상태 유지"**를 재확인해 줌.
* **최종 결론**:
  * 복잡한 설계를 모두 전면 폐기하고 가장 직관적이고 심플한 구조로 회귀.
  * `stock_build.yml` 내의 Pages 배포 스텝은 그대로 **유지**하되, 피처 브랜치 실행을 차단하기 위해 **`on.push.branches: [main]` 트리거 제약조건만 단독 추가**.
  * 이로써 피처 브랜치의 위협은 0%가 되었고, 주식/뉴스 빌드 즉시 두 배포 플랫폼이 최신 `main` 기준으로 동시 갱신되어 두 사이트의 내용이 완벽하게 일치하게 됨.

## 3. 심층 아키텍처 질문 답변 정리

### ❓ Q1. 뉴스/시황 데이터 생성 및 발송 단계를 나눈 레이어 구조는 적정한가?
* **A**: 소프트웨어 공학의 **단일 책임 원칙(SRP)**을 아주 훌륭히 만족하는 모범적 설계입니다.
  * **수집-md생성**: 외부 데이터 갱신에 독립적으로 작동하여 데이터 유실 방지.
  * **html 빌드**: 로컬 md 소스만으로 초고속 뷰 컴파일이 가능하여 디자인 수정 생산성 극대화.
  * **메일/외부 발송**: 채널 장행 시에도 이미 아카이빙된 기록물은 보존되어 높은 안전성 제공.

### ❓ Q2. 왜 뉴스는 `reports/`와 `publish/` 아래에 각각 다른 JSON 파일이 있는가?
* **A**: 두 JSON은 완전히 다른 스키마와 목적을 가진 파일입니다.
  * **`reports/news_*.json` (원천 소스)**: AI가 뉴스 수집 시점에 분석해 준 종합 통계/트렌드/이슈 목록을 담은 150줄가량의 컴파일 보조 소스 파일입니다.
  * **`publish/news/*.json` (서빙 데이터)**: 수집된 수십 건 기사들의 아웃링크 리스트로, 용량이 너무 커서 웹 브라우저의 초기 로딩 성능 최적화를 위해 날짜별로 쪼갠 Lazy-load(지연 로딩) 대상입니다.
  * **주식의 경우**: 주식은 데이터가 가볍고 단순하여 굳이 JSON 원천 소스를 남기지 않고 마크다운 텍스트 파싱으로 충분하여, 오직 통합 서빙 데이터인 `publish/stock/stock-data.json` 단 하나로만 서빙됩니다.

## 4. UI 렌더링 결함 수정 (마크다운 파서 적용)

* **문제 상황**: 
  * `esc(summary)` 함수를 통해 단순 텍스트 처리를 수행한 후 일반 `<p>` 태그에 삽입하여 줄바꿈 뭉개짐 및 볼드체 표기 오류가 발생함.
* **해결 방법**:
  * 사이트 내부 소스에 이미 탑재된 마크다운 컴파일러(`marked.js`)와 헬퍼 함수(`md2html`)를 발견하여 즉각 매핑함.
* **수정 내역** (`publish/app.html` 및 `publish/index.html`):
  ```javascript
  // AS-IS (평문 출력)
  ${summary ? `<p style="font-size:.88rem;margin-bottom:12px">${esc(summary)}</p>` : ""}

  // TO-BE (마크다운 파싱 및 div 블록 요소 렌더링)
  ${summary ? `<div class="summary-md" style="font-size:.88rem;margin-bottom:12px;line-height:1.6">${md2html(summary)}</div>` : ""}
  ```
* **결과**: 줄바꿈이 동그란 불릿 리스트(`<li>`)로 자동 분리되고, 볼드체 강조(`<strong>`) 스타일이 아름답게 입혀져 가독성이 획기적으로 개선되었습니다.
