# Enterprise Profile (대형/실거래 운영 모음)

이 프로필은 실거래 위험이 있거나, 대형 아키텍처 확장이 요구되는 고리스크/고가용성 프로젝트에 적용되는 지침입니다. (예: `kis-trading`)

---

## 1. 거버넌스 및 문서화 규칙
* **엄격한 이행 절차**: 작업 착수 전 [spec.md](./docs/spec.md)의 엄격한 상호 검증, 릴리즈 체크리스트 작성을 의무화합니다.
* **이슈 격리**: 서브에이전트 라우팅 권한 계층([orchestration.md](./agent/orchestration.md))을 적용하여 각 서브에이전트가 지정된 디렉토리 바운더리 밖의 코드를 무단 수정하지 못하게 엄격 통제합니다.

---

## 2. 보안 및 리스크 제어 (금융 및 실거래 철칙)
* **크레덴셜 보호**: API Keys, Secrets, `kis_credentials.json` 등 어떠한 기밀도 코드 내에 하드코딩하지 않으며 `.env` 또는 OS 환경변수를 통해서만 획득합니다.
* **실거래 드라이런 의무화**: 금융/주식 실거래 주문 로직 변경 시, 실전 매매를 유발하는 API 호출은 즉각 금지하고 반드시 Dry-Run(모의 매매) 모드로 수집 검증한 후, 사용자 최종 Proceed 승인 하에서만 실거래 통로를 개방합니다.
* **롤백 시나리오**: 코드 변경으로 인한 오동작 발생 시, 직전 정상 커밋으로 안전하게 되돌릴 수 있는 Git 기반 롤백 시나리오를 설계서에 의무 기재합니다.

---

## 3. 품질 및 아키텍처 고도화
* **Strict Type-Checking**: TypeScript strict mode 및 Python의 Mypy 정적 검증 기준을 완벽하게 통과해야 합니다.
* **성능 및 비용 거버넌스**: 무한 루프 API 호출 차단, 불필요한 LLM 토큰 낭비 방지를 위한 지식 저장소 분할 등 가용 자원 비용 제어 정책을 준수합니다.
