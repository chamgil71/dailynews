# React & Streamlit 프론트엔드 모범 패턴 가이드 (frontend_react_patterns.md)

이 가이드는 내부 분석 UI(Streamlit)와 외부 서비스(React) 웹을 설계할 때 프론트엔드 코드의 일관성과 렌더링 성능을 지키기 위한 규칙을 정의합니다.

---

## 1. Streamlit 개발 패턴
*   **상태 격리 (Session State)**: Streamlit은 상호작용마다 전체 스크립트가 재실행되므로, 불필요한 상태 손실을 막기 위해 `st.session_state`를 엄격하게 구조화하여 관리합니다.
*   **컴포넌트 함수화**: 화면 배치는 재사용할 수 있도록 `components/` 디렉토리에 함수 혹은 모듈 단위로 작성하여 `app.py`를 간결하게 유지합니다.
*   **캐싱 데코레이터 적용**:
    *   API 호출 결과나 DB 조회는 `@st.cache_data`를 활용합니다.
    *   데이터베이스 커넥션 생성 등 한 번 로드되어 유지되어야 하는 리소스는 `@st.cache_resource`를 사용합니다.

## 2. React 개발 패턴
*   **커스텀 훅 (Custom Hooks)**: UI 프레젠테이션 컴포넌트 내부에서 직접적인 API 통신이나 복잡한 상태 가공 로직을 직접 구현하지 말고, `hooks/useFetchStocks.ts` 같이 별도의 훅으로 모듈화하여 분리합니다.
*   **컴포넌트 크기 통제**: 컴포넌트가 500라인을 초과하거나 15개 이상의 하위 컴포넌트를 직접 렌더링하는 경우, 책임을 쪼개어 서브 컴포넌트로 분리합니다. (God Component 방지)
*   **리소스 중단**: API 호출 비동기 훅을 선언할 때, 마운트 해제 시 호출을 중단하기 위한 `AbortController` 취소 처리를 필수로 포함합니다.
