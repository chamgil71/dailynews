# 스토리지 관리 및 파일 포맷 물리 명세 표준 (storage_standards.md)

본 문서는 프로젝트 영속성(Persistence) 계층 중 파일 시스템 스토리지(`storage/`)의 내부 디렉토리 구조 표준과 데이터 형식별 파일 확장자 및 저장 규칙을 규정합니다.

---

## 1. 스토리지 5대 물리 서브디렉토리 표준

프로젝트 루트의 `storage/` 디렉토리는 성격이 다른 파일들이 혼재되어 유실되거나 덮어쓰여지는 사고를 막기 위해, 에이전트 구동 시 즉시 다음 5대 물리 서브디렉토리 구조를 자동 감지하거나 생성해야 합니다.

```text
storage/
├─ data/        # 로우 데이터, 수집 데이터, 입력 데이터 저장
├─ logs/        # 시스템 실행 로그, API 통신 로그 보관
├─ artifacts/   # AI 모델 출력물, 최종 분석 보고서(Markdown/PDF) 보관
├─ cache/       # API 응답 임시 캐시 및 파싱 결과 보관
└─ exports/     # 최종 내보내기용 산출물(CSV, Excel 등) 보관
```

*   **규칙**: 스토리지 디렉토리에 파일을 기록할 때는 목적에 따라 반드시 위의 5개 서브디렉토리 중 하나를 경로에 명시해야 합니다. 루트 `storage/` 바로 아래에 파일을 난잡하게 생성하는 것을 절대 금지합니다.

---

## 2. 데이터 형식별 표준 파일 포맷 매핑 규칙

생성 및 저장하는 데이터의 성격과 용량에 따라 매핑해야 할 표준 파일 포맷 확장자 약정입니다.

| 데이터 유형 | 권장 파일 확장자 | 사용처 및 규칙 |
| :--- | :---: | :--- |
| **API 요청/응답 & 설정** | `.json` | 구조화된 경량 데이터 저장, 인코딩은 항상 `utf-8` 강제. |
| **정형 및 통계 데이터** | `.csv` | 표 형식의 정형 분석 데이터 저장. 판다스(`pandas`) 이용 I/O. |
| **인간 가독성 보고서** | `.md` | 에이전트 수행 결과 보고서 및 요약본 작성 표준. |
| **대용량 정형 데이터** | `.parquet` | 10MB 이상의 대용량 테이블형 데이터 저장 시 압축 및 성능 최적화용. |
| **Python 인메모리 객체** | `.pickle` | Python 객체의 일시적인 직렬화/역직렬화 용도 (외부 공유 금지). |

---

## 3. 재현성 보장 및 덮어쓰기 방지 원칙

*   **타임스탬프 또는 버저닝 의무화**:
    *   기존 파일을 무조건 덮어쓰거나 무작위 이름을 부여하는 것을 방지하기 위해, 파일명에는 항상 갱신 일자(`YYYY_MM_DD`) 또는 버전을 접미사로 포함합니다.
    *   *좋은 예*: `storage/artifacts/portfolio_analysis_2026_06_26.md`
    *   *나쁜 예*: `storage/artifacts/analysis.md`
*   **임포트 금지**:
    *   `storage/` 디렉토리 하위의 파일들은 실행 결과물 저장 전용이므로, 프로젝트 소스코드 내부에서 코드 패키지처럼 `import`하여 논리적 의존성을 맺어서는 안 됩니다. (단방향 저장만 허용)
