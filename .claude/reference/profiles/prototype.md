# Prototype Profile (소규모/실험 모드)

이 프로필은 단일 HTML, 단순 스크립트, 혹은 임시 인큐베이터 실험 단계의 소규모 프로젝트에 적용되는 경량화 에이전틱 코딩 지침입니다.

---

## 1. 거버넌스 및 문서화 규칙
* **최소한의 문서화**: 무거운 PRD나 설계 명세 대신, [spec.md](../../../docs/spec.md) 하나에 핵심 기능 요구사항만 정리하고 바로 개발을 시작합니다.
* **작업 로그 경량화**: `worklog.md`를 의무화하지 않으며, 작업 완료 후 주요 구현 특이사항만 README에 가볍게 기재합니다.

---

## 2. 아키텍처 원칙 (백엔드 서버 도입 금지)
* **단일 구조 지향**: 별도의 API 백엔드 서버(FastAPI 등)를 구축하지 않고 Frontend 단일 계층(HTML + Vanilla JS 등)에서 직접 처리하는 것을 원칙으로 합니다.
* **데이터 보존**: 로컬 파일 저장이나 browser LocalStorage 또는 JSON 캐시 파일을 우선 사용합니다.

---

## 3. 품질 및 검증
* **자동화 테스트 비필수**: pytest나 Vitest 등의 자동화 테스트 구축을 강제하지 않으며, 최종 결과 작동성에 대한 수동 확인으로 완료 검증을 마칩니다.
* **외과 수술식 변경**: 요청된 범위 내에서만 변경을 집행하여 오버 엔지니어링을 철저하게 방지합니다.
